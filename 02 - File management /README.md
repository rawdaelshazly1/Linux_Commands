# 02 - File Management

## Objective

In this chapter, I learned how to manage files and directories using Linux commands and practiced them in the terminal.

## Commands I Learned

- cp – Copy files and directories.
- mv – Move or rename files.
- rm – Remove files.
- rm -r – Remove directories and their contents.
- echo – Write text into a file.
- > – Create or overwrite file content.
- >> – Add new content to the end of a file.

## Practice

I practiced basic file management by creating files and editing their contents.

I also used:
- cp to copy files.
- mv to move and rename files.
- rm to delete files.
- echo with > and >> to create and update log files.

## What I Learned

- The difference between copying and moving files.
- rm -r should be used carefully because it deletes directories.
- > replaces the old file content.
- >> adds new content without removing the existing content.

## Notes

I noticed that using > can overwrite the file, so I need to be careful. If I only want to add new text, I should use >> instead.
